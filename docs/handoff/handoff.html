HANDOFF · PROTOTYPE → ONTOGONY-SITE

From the prototype to the repo.

Five completed templates, mapped onto the existing Astro project. This sheet specifies the token migration, the component map (what to keep, what to add, what to retire), and the proposed extension to the concept content schema. The accent system is gold + blue; the type system gains Source Serif 4 alongside JetBrains Mono.

§ 1The five completed templates

Each is a static HTML file in this prototype, ready to be ported as an Astro layout / page. Open them and pull markup directly.

  • 01 Homepage Vertical stratified map: hero → R4 / R3 / R2 / R1 bands → reading paths → colophon. Stratum locator follows scroll. → src/pages/index.astro
  • 02 Concepts atlas (index) All concepts grouped by stratum, alphabetical within. Same band rhythm as the homepage; tighter row component. → src/pages/concepts/index.astro
  • 03 Concept detail Ledger of fixed fields (Register, Definition, Burn-rate, Cessation signature, Genealogy, Related, What it is not, Where to go next), then short prose, then a closing “Where to go next” block. → src/pages/concepts/[slug].astro
  • 04 Essay detail Long-form prose: standfirst, body with footnote markers, pull-quote rule, marginal notes, then “Where to go next”. → src/pages/essays/[slug].astro
  • 05 Reading path Atlas-route component: numbered stops, gold spine on the left, blue cursor at the active step, terminal end-glyph, plus a sticky path ribbon at viewport bottom. → src/pages/paths/[slug].astro

UX invariant on every page

Every template ends with a Where to go next block (3–5 forward moves, kind-tagged). This answers the brief's central UX question and gives the dev a single shared component to ship: <WhereNext steps={[…]} />.

§ 2Token migration

The repo's src/styles/global.css currently ships six color tokens, two font tokens, and one accent. The prototype expands this to the field-guide system. Migration is additive: keep the repo's existing names, add the new ones, retune the values.

2.1 — Color tokens

Repo (current) Prototype value Role Status
--color-paper oklch(96% 0.012 85) Default page background. Warm ivory. Retune
— add — --color-paper-2 Tinted band background (alternating R-bands). Add
--color-ink oklch(22% 0.012 60) Body text, headings. Retune
--color-ink-soft oklch(38% 0.010 60) Standfirst, glosses, secondary prose. Retune
--color-ink-muted oklch(56% 0.008 60) Metadata, mono labels, footnote markers. Retune
--color-rule oklch(78% 0.006 70) Hairline dividers. Default underline. Retune
— add — --color-rule-strong Section openers, ledger top-rule. Add
--color-accent (was #6b3a1f rust) oklch(70% 0.10 80) Register tags, stratum markers, paid/unpriced badges, route spine. Migrate the existing variable's value from rust to muted gold; keep the name. Retune
— add — --color-accent-soft Tag underlines, route-spine background. Add
— add — --color-active The transduction color. Used only for active state: link hover, current path step, “you are here”. Add

The two-accent split is doing real semantic work. Gold is structural — it labels what is. Blue is kinetic — it labels where you are. Do not let the dev collapse them back to a single accent.

2.2 — Type tokens

Repo (current) Prototype Status
Aa --font-serif: Crimson Pro --font-serif: Source Serif 4 Replace
Aa --font-sans: Inter --font-sans: Inter Tight Replace
Aa — missing — --font-mono: JetBrains Mono Add

The mono is the load-bearing addition. Without it, register tags, instrument readouts, the locator, footnote markers, and ledger field labels all collapse into the sans, which flattens the field-guide texture. Adding mono is non-negotiable.

§ 3Component map

The repo started a small port already. Here's exactly what each existing class maps to, what to add, and what is retired by the new system.

Repo class Prototype class Action Notes
.page-shell .frame Replace Three-column grid (left rail / reading / right rail) instead of single-column shell. Falls back to single column on narrow viewports.
.kicker .micro Rename Same role; mono instead of sans for the field-guide texture.
.page-title / .home-title .h1 / .display Keep both .display for hero only; .h1 for everything else. Both serif, weight 400 (not 600 — Source Serif 4 holds presence at 400).
.lede .standfirst Rename Field-guide vocabulary. Roman serif (no italic) at 22px; italic only on the actual page-by-page instances when they want it.
.prose .body Keep + extend Adds: p + p indent, sup.fn footnote markers, p.lede opening paragraph. Width tightened to 58–68ch for reading.
.next-steps-list .where-next Rename Same intent; the prototype version uses title / kind / arrow rather than the repo's kind / title. Either layout works — pick one.
.ledger-list / .ledger-row .ledger / .ledger-row Diverge Two different uses. Repo's ledger-list is a list-of-concepts component. Prototype's ledger is the fixed-field grid inside a single concept page (Register / Definition / Burn-rate …). Both are needed; rename one. Suggested: rename repo's to .atlas-list (it's a list of atlas entries); reserve .ledger for the fixed-field component.
.ledger-meta .ledger-row[Related] Fold in The repo's .ledger-meta (label + values list) is just one row of the prototype's full ledger. Implement it as a single row with a comma-separated values list inside <dd>.
.instrument-bar .instrument-bar Diverge Repo's is a top nav (wordmark + UPPERCASE links). Prototype's is wordmark + page-context readouts (kind, register, position) — links live in the left rail. Suggested: adopt the prototype's split, since the field guide has more hierarchy than nav can carry.
— missing — .nav Add The actual site navigation. Mono, vertical, in the left rail. Links: Home / Theory / Concepts / Essays / Paths / Diagrams / About.
— missing — .stratum-locator Add The R4 / R3 / R2 / R1 ladder in the left rail. aria-current="true" gilds the active stratum. On the homepage, an IntersectionObserver tracks scroll across the bands.
— missing — .band Add Section component used on Home and Concepts atlas. Top rule, header (stratum tag + name + gloss + count), then list. Optional .tinted modifier for alternating bands.
— missing — .anchor-list / .atlas-list Add Two list flavours. .anchor-list for the homepage (3 columns: title / gloss / meta, bigger type). .atlas-list for the concepts index (4 columns: name / gloss / register-tag / arrow, smaller type).
— missing — .route / .route-stop / .route-end Add The reading-path component. Vertical gold spine, dot per stop, blue dot for the active stop, diamond for the terminal step. Pure CSS — no SVG.
— missing — .path-ribbon Add Sticky bottom strip, only present when a reading path is active. Mono. Renders the full route inline; current step is blue. Implementation suggestion below.
— missing — .tag Add The register chip: R1 R2 R3 R4. Mono, gold, with optional gold underline. The single most common micro-element across the system.
— missing — .pullquote Add Essay component: italic serif, 22px, bordered top + bottom by hairlines. Used sparingly.
— missing — .section-glyph Add A centered character (·, §, ❧) flanked by hairlines. Replaces ordinary <hr> between major essay sections.
— missing — .colophon Add Footer. Three columns of small sans text: links / colophon paragraph / correspondence.

§ 4Schema extensions

The repo's concept schema is currently { title, short, order?, related[] }. The MDX bodies already manually contain a What it is not section and inline forward references — proof that the structure exists in the prose. Lifting them into frontmatter unlocks the full ledger as a layout, not as repeated authorial discipline.

4.1 — Concepts

src/content.config.ts (excerpt) 
const concepts = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/concepts' }),
  schema: z.object({
    // existing
    title:    z.string(),
    short:    z.string(),
    order:    z.number().optional(),
    related:  z.array(z.string()).default([]),

    // new — structural fields exposed to the ledger
    register:   z.enum(['R1', 'R2', 'R3', 'R4']),                // stratum locator + tag
    chapter:    z.string().optional(),                          // e.g. "Ch. 7 §3.2"
    burnRate:   z.string().optional(),                          // prose, denominated
    cessation:  z.string().optional(),                          // the cessation signature, prose
    genealogy:  z.array(z.string()).default([]),                 // e.g. ["Simondon 1958", "Friston 2010"]
    notThis:    z.array(z.string()).default([]),                 // "Not a phase transition that…", etc
    whereNext:  z.array(z.object({
      kind: z.enum(['concept', 'essay', 'path', 'diagram']),
      slug: z.string(),
      title: z.string().optional(),
      why: z.string().optional(),                                 // the gloss under the link
    })).default([]),
  }),
});

The register field becomes required. It's the spine of the whole macro-organisation — without it the homepage and atlas can't sort.

4.2 — Essays

essays 
const essays = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/essays' }),
  schema: z.object({
    title:       z.string(),
    summary:     z.string(),
    date:        z.coerce.date().optional(),
    register:    z.enum(['R1', 'R2', 'R3', 'R4']).optional(),  // primary stratum
    readingTime: z.number().optional(),                         // minutes; printed in readouts
    number:      z.number().optional(),                         // "Essay 03"
    whereNext:   z.array(z.object({
      kind: z.enum(['concept', 'essay', 'path', 'diagram']),
      slug: z.string(),
      title: z.string().optional(),
      why: z.string().optional(),
    })).default([]),
  }),
});

4.3 — Reading paths

paths 
const pathStop = z.object({
  kind:   z.enum(['concept', 'essay', 'note']),
  slug:   z.string().optional(),                                  // for concept | essay
  title:  z.string().optional(),                                  // override or note title
  body:   z.string().optional(),                                  // inline prose for "note" kind
  register: z.enum(['R1', 'R2', 'R3', 'R4']).optional(),
  minutes:  z.number().optional(),
});

const paths = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/paths' }),
  schema: z.object({
    title:       z.string(),
    description: z.string(),
    number:      z.number().optional(),                         // "Path 02"
    strataCovered: z.array(z.enum(['R1','R2','R3','R4'])).default([]),
    totalMinutes: z.number().optional(),
    steps:       z.array(pathStop),
  }),
});

Every page template the prototype ships uses these new fields. Until they exist in the schema, the layout has to either hardcode them in MDX or punt to a default — both worse than just extending the schema.

§ 5Type rules

5.1 — Three families, three jobs

Family Used for Never used for
Source Serif 4 Headings, body prose, ledger values, route stop titles. Metadata, navigation, register tags.
Inter Tight Footer, colophon. Used minimally. Body. Headings.
JetBrains Mono Register tags, instrument readouts, navigation labels, ledger field labels, footnote markers, page numbers, kind tags, stratum locator. Body prose. Headings.

5.2 — Scale (canonical density)

TokenSize / lineUsed for
.display52 / 1.05Hero on homepage, occasionally elsewhere.
.h138 / 1.10Page title (concept, essay, path).
.h226 / 1.20Section heads inside body prose.
.standfirst22 / 1.45Roman serif standfirst directly below title.
.body p18.5 / 1.65Reading text.
.label12 monoSection labels, ledger keys.

Three densities (Generous / Canonical / Dense) are exposed as a Tweak in the prototype. Ship Canonical as default; the others are data-density="…" on <body>.

§ 6Tweaks (already approved)

Three Tweak axes are wired into the prototype. They persist via the standard __edit_mode_set_keys protocol against an EDITMODE-BEGIN/END JSON block in index.html. For the Astro port, lift them to a small client-only React island, or cookie-back them and apply via a data-* attribute on <body> in the layout.

  • Stratification posture — Descend (default) / Ascend / Level. Flips the R-stack and de-emphasises strata when level. data-posture.
  • Reading temperature — Ivory (default) / Bone / Graphite. Three palette retunes; Graphite is the inverted nighttime variant. data-temp.
  • Field density — Generous / Canonical (default) / Dense. Re-paces the page at once. data-density.

§ 7Open questions

  • Theory page (Core theory). Not yet designed. Proposal: a single long-scroll essay with the same body component as essays, plus a sticky table-of-contents in the right rail (replacing the margin notes for that page only).
  • Diagrams page. Out of scope per the latest brief, but the homepage links to it. Proposal for the placeholder: a simple atlas-list of figures, each with a caption and a chapter cite. No interactive diagrams (per brief).
  • Footnotes. Currently inline-anchor with manual sup.fn markers. Proposal: a remark/rehype plugin to lift Markdown footnote syntax ([^1]) into sup.fn automatically and render the footnote block in the right margin where space permits, falling back to end of essay.
  • Search. Not in the prototype. The atlas-list and stratum-locator carry most of the discovery load. If search is added, propose a /-triggered command bar that searches concept titles + glosses only.
  • Path ribbon persistence. The ribbon currently appears on path-bound pages. Behaviour when a user clicks out to an unrelated concept: should the ribbon persist (with a "leave path" affordance) or vanish? The prototype persists; recommend the dev keeps that.
