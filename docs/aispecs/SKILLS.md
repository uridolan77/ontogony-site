# Ontogony AI Space Skills

## 1. Book-to-Site Translation

Help translate the book into a navigable website.

This means identifying:

- core concepts
- chapter problems
- anchor scenes
- conceptual operators
- diagrams
- reader paths
- chapter-to-chapter handoffs

The goal is to make the book’s architecture visible without flattening the book.

## 2. Chapter Companion Writing

Turn dense chapter material into site-facing companion pages.

A companion page should preserve the chapter’s intellectual force while making its role in the whole book clear.

Useful structure:

```md
## The problem

## The opening ordeal

## The operator

## Key concepts

## What this chapter establishes

## What it hands forward

## Read next
```

## 3. Concept Extraction

Extract field-guide entries from manuscript chapters.

A concept entry should be usable independently, but should also point back into the book’s sequence.

Each concept should ideally include:

- short definition
- expanded definition
- chapter source
- register / stratum
- related concepts
- what it is not
- where next

## 4. Reading Path Design

Create routes through the book-site.

A reading path can follow:

- the book sequence
- a philosophical theme
- an AI/knowledge-system theme
- a reader type
- a conceptual problem

Paths should feel like guided traversals through the theory.

## 5. Diagram Thinking

Transform chapter structures into diagrams.

Good diagram targets:

- the book spine
- R1–R4 strata
- Variation / Encounter / Mediation
- Genesis Engine
- Stabilisation Engine
- Stratification Engine
- Witness–Canon architecture
- cognitive strata
- symbolic exit
- sixth transduction

## 6. Editorial Stewardship

Help keep the site aligned with the book as the manuscript evolves.

This includes:

- naming consistency
- title consistency
- concept consistency
- chapter references
- where-next links
- updated summaries
- navigation coherence

## 7. Technical Stewardship

Understand the current architecture:

- Astro
- MDX
- Content Collections
- Tailwind
- Vercel
- GitHub
- future TinaCMS

Help implement changes without losing the MDX-in-Git source of truth.

## 8. CMS / Workflow Design

Help design a CMS workflow that supports the book.

The CMS should make it easier to manage:

- chapter companion essays
- concept pages
- diagrams
- fragments
- reading paths
- draft/published status
- chapter links
- related concepts

## 9. Content Voice

Preserve the book’s tone:

- precise
- serious
- argument-driven
- materially grounded
- philosophical
- diagnostic
- vivid when anchored in an ordeal
- clear enough for a public reader

## 10. Site Identity

Preserve the visual and structural identity:

```txt
Atlas
Ledger
Instrument
Field guide
Book companion
Chapter map
Concept ledger
Reading route
```
